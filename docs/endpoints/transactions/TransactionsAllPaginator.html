Title

GET /DneroArk/transactions

Receives a list of transactions, starting from the most recent to the oldest, with pagination support.

If no parameters are provided, it returns all notifications from the most recent to the oldest. If we want to use the paginator, we can include the parameters.

Header​

KeyValue
Acceptapplication/json
AuthorizationBearer key

Parameter​

Paginator parameters

NameTypeDescription
pageSizeintThe number of items to include in each page of the results. Required if using the paginator.
pageintThe page number to retrieve, starting from 1 for the first page. Required if using the paginator.

Query parameters example

/DneroArk/transactions?pageSize=10&page=2

Status and Paginator parameters

NameTypeDescription
statusesint[]An array of statuses to filter the transactions. If null or 0, all statuses are returned. Multiple statuses can be provided.
pageSizeintThe number of items to include in each page of the results. Required if using the paginator.
pageintThe page number to retrieve, starting from 1 for the first page. Required if using the paginator.

Query parameters example

/DneroArk/transactions?statuses=1,2&pageSize=10&page=3

JSON Response: Success (200)​

List of notifications with pagination

NameTypeRequiredDescription
currentPageint✅The current page number being returned (starting from 1 for the first page).
totalPagesint✅The total number of pages available based on the pageSize and total records.
pageSizeint✅The number of items per page in the paginated results.
totalTransactionsstring✅The total number of transactions available based on the provided filters.
transactionsArray✅A list containing all the transactions for the requested page.
idlong✅Unique identifier of the transaction notification.
typestring✅Possible values: sent or received.
userNamestring✅If type is sent, it’s the recipient's name; if received, it’s the sender's name.
userImgUrlstring✅If type is sent, it’s the recipient's image Url; if received, it’s the sender's image Url.
dateDateTime✅The date and time when the transaction was created.
amountstring✅The amount of money involved in the transaction.
coinStatusint✅Coin status must not be zero. For detailed information, see the Status Codes page.
expirationDateDateTime❌Expiration date of the coin; if it does not expire, it should be null.
capturedDateDateTime❌Date when the coin was captured; if it has not been captured, it should be null.
